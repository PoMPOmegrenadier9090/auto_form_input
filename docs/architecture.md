# Architecture Guide

This document describes the current v1 architecture of AI Auto Form Filler and the rules for safe future changes.

## 1. High-Level Flow

1. User starts detection from popup.
2. Content script detects/collects form containers.
3. Content script builds locator map and sanitized HTML.
4. Background script builds prompts and calls LLM provider.
5. LLM response is parsed into operation `steps`.
6. Content script executes steps to resolve values and fill DOM fields.
7. Overlay/popup show completion summary and confidence buckets.

## 3. Operation Contract

The only accepted LLM output contract is:

- `steps: InputPlanStep[]`
- `unmapped: string[]`

`InputPlanStep` is defined in [types/index.ts](../types/index.ts) and currently supports:

- `direct`
- `concat`
- `template`
- `select_match`
- `split`

### 3.1 Adding a New Operation

Always update in this exact order:

1. Type definitions in [types/index.ts](../types/index.ts)
2. Prompt instructions/examples in [lib/llm/prompt-builder.ts](../lib/llm/prompt-builder.ts)
3. Parser validation in [lib/llm/response-parser.ts](../lib/llm/response-parser.ts)
4. Execution in [lib/form-filler.ts](../lib/form-filler.ts)
5. Build + manual validation

## 4. Ref and Locator Stability

- Refs are unique across containers using prefix format (e.g. `c1-3`, `c2-1`).
- LLM must treat ref as an opaque token.
- Locators are generated by priority: `id` -> `name+type CSS` -> unique CSS -> XPath.

## 5. Data Privacy Model

- Public fields (`isPublic=true`): value can be sent to LLM.
- Private fields (`isPublic=false`): real value is not sent.
- Dummy values (`dummyValue`) can be sent for format understanding.

### 5.1 Prompt Data Minimization

Sanitized HTML sent to LLM intentionally omits runtime user-entered state:

- `value` attribute is removed.
- `checked` / `selected` states are removed.
- `textarea` text content is cleared.

This reduces both leakage risk and token usage.

### 5.2 Excluded Input Types

The shared selector excludes high-risk/non-fill targets:

- `hidden`
- `password`
- `file`
- `submit`
- `button`
- `reset`
- `image`

See [lib/selectors.ts](../lib/selectors.ts).

## 6. Logging Policy

All debug logs should go through [lib/debug.ts](../lib/debug.ts).

- Default: `DEBUG_AUTOFILL_LOGS = false`
- Enable logs only when investigating issues.

Do not add direct `console.log` calls outside the debug helper.

## 7. Prompt Size Estimation

Content script estimates prompt size from sanitized HTML and publishes to UI:

- `htmlCharCount`
- `htmlWordCount`
- `estimatedTokenCount` (rough approximation)
- `tokenWarning`

This helps avoid context window overflow.

## 8. Provider Layer

Provider selection is configured in options:

- OpenAI
- Gemini

Provider factory: [lib/llm/provider.ts](../lib/llm/provider.ts)

Each provider must return text parseable by [lib/llm/response-parser.ts](../lib/llm/response-parser.ts).

### 8.1 Provider Auth Notes

- OpenAI: Bearer token in `Authorization` header.
- Gemini: API key in `x-goog-api-key` header (not query string).

## 9. Versioning Rule

Current baseline is **v1**.

- No backward compatibility branch for legacy mapping format.
- Changes that alter operation schema should be documented in this file and reflected in README.
